Chương 2 — Mô hình Ngôn ngữ Lớn

Tổng quan LLM

LLM là foundation models — huấn luyện trên hàng nghìn tỷ token
3 giai đoạn phát triển: Transformer (2017) → GPT-3 (2020) → GPT-4, LLaMA 3, Qwen 3.5 (2023+)
Kiến trúc chủ đạo: Decoder-only Transformer với causal mask
3 giai đoạn huấn luyện: Pre-training → SFT → Alignment (RLHF/DPO)
GPT-4 ~1.8T tham số LLaMA 3.1 405B Qwen 3.5 128K ctx Gemini 1.5 1M ctx DeepSeek-V3 MoE
07

Chương 2

Kiến trúc Transformer

Encoder (N=6)

Multi-Head Self-Attention

→ Add & Norm

→ Feed Forward

→ Add & Norm

Decoder (N=6)

Masked Multi-Head Self-Attention

→ Add & Norm

→ Cross-Attention (từ Encoder)

→ Feed Forward → Linear → Softmax

Đặc điểm: Self-Attention × Multi-Head × Position Encoding

08

Chương 2

Cơ chế Self-Attention

Attention(Q,K,V) = softmax(QKᵀ/√dₖ) × V

Q (Query): "token này đang tìm gì?"
K (Key): "token này chứa thông tin gì?"
V (Value): thông tin thực tế được truyền đi
Multi-Head: 8 head song song, mỗi head học 1 kiểu quan hệ khác nhau

Chương 3 — Truy vấn Tạo sinh Tăng cường

Tổng quan RAG

Ý tưởng: Truy xuất tài liệu liên quan → cung cấp làm ngữ cảnh cho LLM
Lợi ích 1: Cập nhật tri thức không cần huấn luyện lại LLM
Lợi ích 2: Giảm hallucination — neo câu trả lời vào tài liệu thực
Lợi ích 3: Minh bạch — trích dẫn nguồn cụ thể (phần, chương, trang)

Lewis et al. (2020) — NeurIPS 2020

Chương 3

Quy trình Naive RAG

1. Indexing

PDF → OCR → Structured Chunking → ChromaDB

2. Embedding

Text → bge-m3 → Vector 1024d

3. Retrieving

Query → Similarity Search → Top-K Chunks
Hybrid: Sparse (BM25) + Dense (Bi-encoder)

4. Generating

Context + Prompt → LLM → Answer (streaming SSE)

Pipeline RAG 3 tầng

Câu hỏi → Query Expansion (từ đồng nghĩa tiếng Việt) → Topic Filter (metadata)
Tầng 1: Bi-encoder Dense Retrieval (bge-m3)  ||  Tầng 2: BM25 Sparse Retrieval
→ RRF Fusion (kết hợp kết quả)
→ Tầng 3: LLM Reranker (chọn top-K cuối)
→ MMR (đa dạng hóa) → Context → LLM → Trả lời + trích dẫn
